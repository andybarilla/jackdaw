# Merge Workflow Design

Merge agent work back to main from within the app. Squash-merges worktree branches after auto-rebasing, with access from both the diff viewer and the cleanup dialog.

## Scope

Worktree sessions only. Non-worktree sessions have no branch to merge. The merge target is always the session's `BaseBranch` (typically `main`).

## Merge Strategy

- **Squash merge** by default. A config setting (`MergeMode`) switches to regular merge.
- **Auto-rebase** before merge: `git rebase <baseBranch>` in the worktree. If conflicts arise, abort the rebase and surface the error — no conflict resolution UI.
- **Auto-generated commit message**: derived from branch name. Strip common prefixes (`jackdaw-`, `feat-`, `fix-`), replace hyphens with spaces, capitalize first letter. Example: `jackdaw-add-merge-workflow` becomes `Add merge workflow`. No edit step.
- **Auto-cleanup** after successful merge: remove worktree and delete branch, then show a toast notification confirming the merge. No confirmation dialog.

## Entry Points

### 1. Cleanup Dialog (primary)

The existing `WorktreeCleanupDialog` gains a third button: **Merge to \<baseBranch\>**. Button order: `[Keep worktree] [Merge to main] [Delete worktree]`. The merge button is styled as the primary/accent action.

The merge button is disabled with a tooltip when:
- There are uncommitted files (tooltip: "Commit or stash changes first")

Unpushed commits are fine — they're what we're merging.

### 2. Diff Viewer

A **Merge** button in the diff viewer's file list header area (next to the file count). Same disabled conditions as the cleanup dialog. Clicking triggers the same merge flow.

## Backend

### New Go function: `worktree.Merge`

```go
type MergeResult struct {
    Success       bool   `json:"success"`
    CommitMessage string `json:"commit_message,omitempty"`
    Error         string `json:"error,omitempty"`
}

func Merge(repoDir, worktreePath, branchName, baseBranch string, squash bool) (*MergeResult, error)
```

Sequence:
1. `git -C <worktreePath> rebase <baseBranch>` — if fails, `git rebase --abort` and return error
2. `git -C <repoDir> checkout <baseBranch>`
3. If squash: `git -C <repoDir> merge --squash <branchName>` then `git -C <repoDir> commit -m "<message>"`
4. If regular: `git -C <repoDir> merge <branchName> -m "<message>"`
5. Return `MergeResult` with success and commit message

The commit message is generated by a helper `commitMessageFromBranch(branchName string) string`.

### New binding: `App.MergeSession`

```go
func (a *App) MergeSession(sessionID string) (*worktree.MergeResult, error)
```

Sequence:
1. Look up session, validate it's a worktree session
2. Load config to determine squash vs regular merge
3. Call `worktree.Merge(info.OriginalDir, info.WorktreePath, info.BranchName, info.BaseBranch, squash)`
4. On success: call `worktree.Remove(...)` to clean up worktree and branch
5. Call `manager.Kill(sessionID)` to remove the session
6. Return result

### Config addition

```go
type Config struct {
    // ... existing fields
    MergeMode string `json:"merge_mode,omitempty"` // "squash" (default) or "merge"
}
```

Default is empty string, treated as `"squash"`.

### App.svelte orchestration

Both entry points call the same async function:

```typescript
async function mergeSession(sessionId: string): Promise<void> {
    const result = await MergeSession(sessionId);
    if (result.success) {
        // Toast: "Merged: <commit_message>"
        // Close cleanup dialog if open
        // Close diff pane if open
    } else {
        // Toast error: result.error
    }
}
```

## Error Handling

- **Rebase conflicts**: `Merge` aborts the rebase and returns an error. Toast shows "Rebase failed — resolve conflicts manually". The worktree and session are untouched.
- **Uncommitted files**: Merge button disabled in UI. Backend also validates and returns error if called directly.
- **Not a worktree session**: `MergeSession` returns error.
- **Worktree removal fails after merge**: The merge itself succeeded. Log the cleanup error but report success to the user. The stale worktree can be cleaned up later.
- **Session already exited**: Merge still works — it operates on the git repo, not the process.
