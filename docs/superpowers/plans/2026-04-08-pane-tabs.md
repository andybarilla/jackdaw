# Pane Tabs — Implementation Plan

Spec: `docs/superpowers/specs/2026-04-08-pane-tabs-design.md`

## Task 1: Update data model and layout helpers

**Files:** `frontend/src/lib/layout.ts`, `frontend/src/lib/layout.test.ts`

Change `LayoutNode` leaf type from `{ type: "leaf"; content: PaneContent }` to `{ type: "leaf"; contents: PaneContent[]; activeIndex: number }`. Remove `null` from `PaneContent` (empty pane = empty array).

Update all layout functions:

- `emptyLeaf()` → `{ type: "leaf", contents: [], activeIndex: 0 }`
- `getLeafContent(node, path)` → returns `contents[activeIndex]` or `null` if empty
- `setLeafContent(node, path, content)` → retained, replaces active tab content
- `findLeafBySessionId` → returns `{ path: Path; tabIndex: number } | null`, searches all tabs in all leaves
- `findLeafByTerminalId` → same return type change
- `findLeafByDiffSessionId` → same return type change
- `collectSessionIds` → collect from all tabs in all leaves
- `collectTerminalIds` → same
- `collectDiffSessionIds` → same
- `collectLeaves` → returns all `PaneContent` items from all tabs in all leaves

Add new functions:

- `addTab(node, path, content)` → appends to `contents[]`, sets `activeIndex` to new tab, returns updated tree
- `removeTab(node, path, tabIndex)` → removes tab, adjusts `activeIndex` (clamp to `contents.length - 1`), returns updated tree
- `setActiveTab(node, path, tabIndex)` → sets `activeIndex`, returns updated tree
- `migrateLayout(node)` → converts old `content` format to new `contents[]` format

Update `splitLeaf` — when splitting, the existing leaf (with all its tabs) stays in `children[0]`, `children[1]` gets `emptyLeaf()`.

Update `unsplitPane` / `collectLeaves` — collect all contents from all tabs.

Update tests to use new data model. Add tests for: `addTab`, `removeTab`, `setActiveTab`, find-by-id across multiple tabs, migration.

## Task 2: Create TabBar component

**Files:** `frontend/src/lib/TabBar.svelte`

New component. Props:
- `contents: PaneContent[]` — the tab items
- `activeIndex: number`
- `sessions: SessionInfo[]` — for resolving session names
- `onSelect: (index: number) => void`
- `onClose: (index: number) => void`
- `onReorder: (fromIndex: number, toIndex: number) => void`

Rendering:
- ~28px height, horizontal strip
- Each tab shows label derived from content type:
  - `session` → `SessionInfo.name` (lookup from sessions prop), truncated with ellipsis
  - `terminal` → "Terminal"
  - `diff` → "Diff: {session name}"
- Active tab has accent-colored bottom border
- X button appears on hover
- Drag-and-drop reordering within the strip (HTML5 drag API)
- Only rendered when `contents.length >= 2`

## Task 4: Update App.svelte — core integration

**Files:** `frontend/src/App.svelte`

Wire up the new tab model throughout App.svelte:

**Sidebar click (`handleSidebarSelect`):**
1. Search all tabs in all leaves for session. If found → focus pane + set `activeIndex` to that tab.
2. Otherwise → `addTab` to focused pane (not `setLeafContent`).

**Session exit (`sessions-updated` handler):**
- When session exits, find its tab (any pane, any tab) and remove it via `removeTab`. If leaf becomes empty, keep as empty pane (quick-picker). Don't collapse.

**Terminal exit (`terminal-exited` handler):**
- Same: find terminal's tab and remove via `removeTab`.

**`handleKill`:**
- Find session's tab and remove it. Don't collapse pane if other tabs remain.

**`handleClosePane` (Ctrl+Shift+W):**
- Kill the **active tab's** process and remove that tab via `removeTab`. If it was the last tab, collapse the pane from layout. If other tabs remain, pane stays.

**`handleUnsplitPane`:**
- `collectLeaves` already updated to return all contents from all tabs — no change needed beyond what Task 1 provides.

**`handleNewSession` / `handleQuickPick`:**
- When assigning to empty pane, use `addTab` instead of `setLeafContent`.

**`session.next` / `session.prev`:**
- When target session is in a background tab, focus the pane AND switch `activeIndex`.

**`openDiffForSession` (`session.viewDiff`):**
- If session is in any tab (active or background), add diff as new tab in same pane (not split).
- If session not in any pane, add diff as new tab in focused pane.

**Tab callbacks (from SplitPane/PaneContainer):**
- `onTabSelect(path, index)` → `setActiveTab`
- `onTabClose(path, index)` → detach tab (remove from contents). If terminal type, kill process. If last tab, pane becomes empty.
- `onTabReorder(path, from, to)` → reorder contents array in leaf

**Startup cleanup:**
- Add `migrateLayout` call before existing cleanup logic.
- Update stale-session/terminal/diff cleanup to iterate all tabs in each leaf.

**Layout persistence:**
- Already works — `contents[]` / `activeIndex` serialize as plain objects.
